Trace A Request End To End

The fastest way to learn SaaS infrastructure is to follow one request until it becomes boring. Every intimidating term is just a device on the path: naming, routing, encrypting, authenticating, authorizing, computing, storing, caching, observing, retrying, and rolling back.

Cross-links: networking, auth, backend design, state, observability.

The Request Path

sequenceDiagram
  participant U as User
  participant B as Browser
  participant DNS as DNS Resolver
  participant CDN as CDN / Edge
  participant LB as Load Balancer
  participant APP as App Server
  participant AUTH as Auth / Session Store
  participant DB as DB / Cache / Queue
  participant OBS as Logs / Metrics / Traces

  U->>B: click / submit / fetch()
  B->>DNS: resolve app.example.com
  DNS-->>B: IP / CNAME chain
  B->>CDN: TCP or QUIC + TLS handshake
  CDN->>LB: HTTP request
  LB->>APP: route to healthy backend
  APP->>AUTH: validate cookie/token/session
  AUTH-->>APP: principal + permissions
  APP->>DB: read/write state
  DB-->>APP: result
  APP->>OBS: trace span + logs + metrics
  APP-->>B: HTTP response + headers
  B-->>U: render updated UI

What Each Hop Decides

HopThe question it answersFailure signature
Browser/appWhat origin, method, headers, cookies, body?CORS error, missing cookie, malformed JSON
DNSWhich IP or edge should receive this name?NXDOMAIN, stale address, slow first request
TCP/QUICCan bytes flow reliably enough?timeout, reset, packet loss, head-of-line blocking
TLSWho am I talking to, and is traffic encrypted?cert error, handshake failure, old protocol
CDN/edgeCan this be cached or blocked at the edge?stale content, WAF false positive
Load balancerWhich healthy backend gets this request?502/503, uneven backend load
App serverWhat business action should happen?500, bad validation, race condition
Auth/sessionWho is this, and what can they do?401, 403, CSRF, confused session
Data/cache/queueWhere is durable or shared state?stale read, deadlock, duplicate job
ObservabilityCan we explain this later?no trace ID, no user-impact metric

Minimal Request Debugging Script

When someone says “the app is down,” ask:

Then split the failure:

  • DNS fails: name resolution problem.
  • TLS fails: certificate, SNI, protocol, trust chain.
  • Edge returns 403: WAF, auth, geo/IP rule, missing header.
  • LB returns 502: backend connection problem.
  • App returns 500: application or dependency problem.
  • App returns 200 but UI broken: frontend asset/API/session/CORS problem.
